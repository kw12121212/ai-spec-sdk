# Proposal: lsp-diagnostics

## What
Implement diagnostic aggregation and reporting for Language Server Protocol (LSP) integrations. This exposes syntax errors, warnings, and hints generated by the connected LSP server to the agent.

## Why
This fulfills the final "Done Criteria" for the `11-lsp-mcp-tools.md` milestone: "LSP diagnostics are surfaced as agent observations". Without this, the agent is blind to code errors that the language server detects, limiting its ability to fix code or understand the current state of the workspace.

## Scope
- Update `LspClient` to listen for and store `textDocument/publishDiagnostics` notifications.
- Implement an `lsp_diagnostics` tool in `lsp-tools.ts` that allows the agent to retrieve the diagnostics for a given file URI.
- Write tests to verify diagnostics are received, stored, and correctly retrieved.

## Unchanged Behavior
- Existing LSP tools (`lsp_hover`, `lsp_definition`, `lsp_references`) remain unchanged.
- The `LspClient` connection and initialization lifecycle remains unchanged.
- The unified tool interface remains unchanged.
